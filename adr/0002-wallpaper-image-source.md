# ADR-0002: 壁紙画像ソースの選定

| 項目 | 内容 |
|------|------|
| ステータス | 承認済み |
| 日付 | 2026-04-01 |
| 意思決定者 | プロジェクトオーナー |

## コンテキスト

天気に連動した壁紙を表示するため、天気条件に合った高品質な画像を動的に取得する仕組みが必要である。

検討した選択肢:

- **Unsplash API** — 高品質なフリー写真のAPI。無料で50リクエスト/時間。商用利用可能。
- **Pexels API** — 同様のフリー写真API。200リクエスト/時間と枠が大きい。
- **ユーザー自前の画像フォルダ** — ユーザーが天気ごとの画像を用意する方式。
- **AI画像生成** — Stable Diffusion等で天気に合った壁紙を生成する。

## 決定

**Unsplash APIを壁紙画像の主要ソースとして採用する。**

## 理由

- Unsplash APIは写真の質が非常に高く、壁紙として適した風景写真が豊富
- ライセンスが明確で、商用・非商用問わず無料で利用可能
- 50リクエスト/時間のレート制限は、天気変化時のみ取得する設計であれば十分
- Pexelsも候補だったが、Unsplashの方が写真のキュレーション品質が高い
- AI画像生成は面白いがローカルでのGPU要件が高く、初期バージョンでは見送り
- ユーザー自前の画像フォルダは将来の拡張案として残す

## 影響

- Unsplash APIのAccess Key取得がセットアップ時に必須となる（ユーザーへの案内が必要）
- レート制限対策として、画像のローカルキャッシュ機構を必ず実装する
- 天気コードごとにデフォルト画像を同梱し、API到達不能時でも動作するようにする
