# ADR-0005: 天気情報オーバーレイの描画方式

| 項目 | 内容 |
|------|------|
| ステータス | 承認済み |
| 日付 | 2026-04-01 |
| 意思決定者 | プロジェクトオーナー |

## コンテキスト

壁紙画像の上に天気情報（気温、天気アイコン、予報など）を表示する方法を決定する必要がある。表示の自由度、パフォーマンス、実装の複雑さを考慮する。

検討した選択肢:

- **Pillow (PIL) で画像に直接描画** — 壁紙画像にテキストを描画し、完成画像を壁紙として設定。
- **PyQtの透明ウィンドウ** — 壁紙の上に透明ウィンドウを常駐させ、テキストを描画。
- **HTML → 画像変換** — HTMLでレイアウトを作りスクリーンショットを取る方式。

## 決定

**Pillowで壁紙画像に直接天気情報を描画する方式を採用する。**

## 理由

- 壁紙画像自体にテキストを焼き込む方式は、追加の常駐プロセスが不要で最もシンプル
- Pillowは軽量でインストールが容易、テキスト描画・半透明合成などの機能が揃っている
- PyQtの透明ウィンドウ方式はCPU常駐負荷がかかり、バッテリー消費の懸念がある
- HTML→画像変換はWebkitやSeleniumの依存が増え、大げさな構成になる
- 壁紙更新は天気変化時のみなので、Pillowでの画像処理コストは実質ゼロに近い

描画仕様:
- 半透明の黒背景ボックス（角丸）を描画し、その上にテキストを配置
- フォントはNoto Sans JP（日本語対応）を推奨、同梱する
- 表示位置はconfig.yamlで四隅から選択可能

## 影響

- フォントファイルの同梱が必要（リポジトリサイズへの影響を考慮し、軽量なフォントを選定）
- テキストの更新のたびに画像処理が走るが、天気変化時のみなので性能影響は軽微
- 将来的にアニメーションエフェクトを追加したい場合は、PyQtオーバーレイへの拡張を検討する
