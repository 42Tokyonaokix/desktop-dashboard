# ADR-0006: 開発言語の選定

| 項目 | 内容 |
|------|------|
| ステータス | 承認済み |
| 日付 | 2026-04-01 |
| 意思決定者 | プロジェクトオーナー |

## コンテキスト

デスクトップ常駐型の天気壁紙チェンジャーを開発するにあたり、開発言語を選定する。開発速度、ライブラリの充実度、macOSとの親和性、OSSとしての配布のしやすさを考慮する。

検討した選択肢:

- **Python** — 豊富なライブラリ、高い開発速度。macOSにプリインストール（ただしバージョンに注意）。
- **Swift / SwiftUI** — macOSネイティブ。最も軽量で高速だが、学習コスト・開発コストが高い。
- **Electron (JavaScript)** — クロスプラットフォーム対応。UIの自由度が高いが、メモリ消費が大きい。
- **Rust** — 高パフォーマンスだが、画像処理・API連携のライブラリがPythonほど充実していない。

## 決定

**Python 3.10+を開発言語として採用する。**

## 理由

- requests、Pillow、BeautifulSoup、PyYAMLなど、本プロジェクトに必要なライブラリがすべて成熟している
- APIコール + 画像処理がメイン処理であり、Pythonの速度で十分（パフォーマンスクリティカルな処理がない）
- OSS配布時、ユーザーがコードを読んでカスタマイズしやすい（Weather Widget Liveのような商用アプリの内部をいじれない不満を解消）
- subprocessによるosascript呼び出しでmacOS壁紙設定が容易に実装可能
- Swiftは高速だが、API連携やHTML解析のコードが冗長になる
- Electronはメモリ常駐で100MB以上消費する可能性があり、壁紙チェンジャーとしてはオーバースペック

## 影響

- ユーザーにPython 3.10+のインストールを求める（pyenvやHomebrewでの案内が必要）
- パッケージ管理にpipを使用し、requirements.txtで依存を明記する
- 将来的にpyinstallerでスタンドアロン実行ファイルにパッケージングすることも可能
