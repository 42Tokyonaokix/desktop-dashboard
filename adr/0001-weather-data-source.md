# ADR-0001: 天気データソースの選定

| 項目 | 内容 |
|------|------|
| ステータス | 承認済み |
| 日付 | 2026-04-01 |
| 意思決定者 | プロジェクトオーナー |

## コンテキスト

天気連動壁紙チェンジャーは現在の天気情報をリアルタイムで取得する必要がある。天気APIは複数の選択肢があり、それぞれ提供データの範囲、コスト、信頼性が異なる。

検討した選択肢:

- **Open-Meteo API** — オープンソースの天気API。完全無料、APIキー不要。気温、天気コード、風速、週間予報などを提供。
- **OpenWeatherMap** — 人気の天気API。無料枠あり（1,000リクエスト/日）だがAPIキー登録必須。データ量は豊富。
- **Yahoo! YOLP 気象情報API** — Yahoo! JAPANが提供。降水強度予測に特化。APIキー要。
- **気象庁API（非公式）** — 日本国内向け。公式APIではないため安定性に不安。

## 決定

**Open-Meteo APIをメインデータソースとし、Yahoo! YOLP APIを降水予測の補完として併用するハイブリッド構成を採用する。**

さらにYahoo天気のWebスクレイピングをフォールバック手段として実装する。

## 理由

- Open-Meteoは完全無料・APIキー不要で、開発・配布のハードルが最も低い
- 天気コード（WMO標準）を返すため、壁紙の切り替えロジックが実装しやすい
- Yahoo YOLPは日本のローカルな降水予測に強く、補完データとして価値が高い
- OpenWeatherMapは無料枠が少なく（1,000リクエスト/日）、10分間隔ポーリングだと1日144リクエストで余裕はあるが、OSSとして配布する際にユーザー全員がAPIキー取得する必要がある点が懸念

## 影響

- Open-Meteoがダウンした場合のフォールバックとしてYahoo天気スクレイピングを実装する必要がある
- Yahoo YOLPのAPIキーはオプション扱いとし、未設定でも動作するよう設計する
- スクレイピングはサイト構造変更で壊れるリスクがあるため、エラーハンドリングを丁寧に実装する
